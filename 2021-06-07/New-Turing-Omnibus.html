Problem 1: Ein Rezept als Algorithmus wiedergeben

Eigentlich denkt man es wäre einfach ein Rezept so wiederzugeben, dass jemand durch diesen Algorithmus, denn ein Rezept ist nichts anderes als ein Algorithmus, genau das gleiche Ergebnis erhält, falls er sich penibel an die Anweisungen hält, wenn er/sie versucht es nachzukochen (ziemlich viele Kommas, Sorry, hatte angekündigt holprig zu sein).

Ich versuche im Nachfolgenden nun ein Rezept zum Kochen eines vegetarischen Nudelauflaufs möglichst genau wiederzugeben. (Bild wird vielleicht nachgereicht.)

Als Zutaten benötigen wir:

  • 500 g Penne Rigate aus Hartweizengrieß

    Hier fängt es schon an. Wie genau muss man sein? Reicht es anzugeben 500 g Nudeln zu benötigen oder müsste man 500 g Penne Rigate aus Hartweizengrieß und die Marke nenne? Am Ende wird kaum einen Unterschied machen, trotzdem gleich wäre es nicht falls 500g Spagetti Nudeln verwendet werden, aber 500 g Penne Rigate aus Hartweizengrieß gefordert wurde.

    Im folgenden werde ich mich darauf beschränken die Menge und den Namen der benötigten Lebensmittel zu nennen (also ohne Marke) . Falls es nötig ist eine spezifischere Angabe zu machen wird sie genannt (z.B.: Nudel wird Penne Rigate).

  • 800 g gehackte Tomaten

  • 200 g Mozzarella

  • 1 Zwiebel

  • 2 Zehen Knoblauch

  • 1 Becher Sahne (200 g)

  • 100 g Parmesan

  • 400 g Tomaten

  • 2 l Wasser

  • 2 EL Olivenöl

Benötigte Requisiten:

  • Schneidemesser
  • Knoblauchpresse
  • Schneidebrett
  • 2 Töpfe
  • Käsereibe
  • Auflaufform

Nun zum eigentlichen Rezept:

  • Ofen auf 200 °C vorheizen

Die folgenden 2 Listen können parallel abgearbeitet werden.

  • Topf mit 2 Litern Wasser füllen, salzen

  • gefüllten Topf auf den Herd stellen, Deckel draufmachen

  • den auf dem Herd stehenden Topf erhitzen und das Wasser zum kochen bringen

  • gelegentlich umrühren

  • Deckel vom Topf nehmen

  • Nudeln ins kochenden Wasser geben

  • Nudeln 10 Minuten Kochen lassen

  • Hitze entfernen

  • Wasser abgießen

  • auf einem Schneidbrett Zwiebel schälen und Knoblauch zerkleinern

  • die Zwiebel und den Knoblauch in zweiten Topf geben

  • zweiten Topf erhitzen

  • zweiten Topf die Zwiebel und den (beide Zehen) Knoblauch mithilfe von 2 EL Olivenöl anbraten

  • 800 g gehackten Tomaten in den zweiten Topf geben

  • gelegentlich umrühren

  • zweiten Topf 4 min köcheln lassen

  • Hitze entfernen

  • den Becher Sahne hinzufügen

  • 100 g Parmesan mithilfe der Käsereibe in den zweiten Topf reiben

  • 400 g Tomaten auf dem Schneidebrett kleinschneiden (halbieren oder vierteln je nach Größe)

  • vermengen

Sind beide Listen abgearbeitet kann mit der nächsten Liste begonnen werden.

  • Inhalt des Nudeltopfs in den zweiten Topf geben und vermengen
  • Inhalt des zweiten Topfs in die Auflaufform geben
  • 200 g Mozzarella auf Schneidbrett in Scheiben schneiden
  • Mozzarella auf der Nudel-Tomatenmenge verteilen
  • Auflaufform in den Ofen stellen
  • 20 Minuten auf mittlerer Schiene backen lassen
  • Hitze entfernen
  • Auflaufform aus dem Ofen nehmen und auf das Schneidebrett stellen
  • Auflauf 10 Minuten abkühlen lassen

Problem 2: Ein gegebenen Algorithmus erweitern

Der im Buch gegebene Algorithmus erzeugt Wallpaper aus einem sich immer wiederholenden Muster. Hier ist der Psudocode:

input corna, cornb
input side
for i ← 1 to 100
	for j ← 1 to 100
		x ← corna + i * side / 100
		y ← cornb + i * side / 100
		c ← int(x²+y²)
		if c even
			then plot(i,j)

Die Aufgabe war nun diesen Algorithmus so zu erweitern, das drei anstelle von zwei Farben verwendet werden. Ich habe mich dazu entschieden die Aufgabe mit C# zu lösen, da ich damit aktuell zumindest am besten zurechtkomme und denke hiermit viele verschiedene Probleme schnell lösen zu können. Im Informatik-Studium haben wir in den letzten Semestern mit Java und C kennengelernt, privat habe ich mir Python angesehen. In Java bin ich mittlerweile auch einigermaßen gut.

// using System;
// using System.Drawing;
// using System.Drawing.Imaging;

Random rnd = new(); // new Random()

// breite und höhe des Bildes
int width = 500;
int height = 500;

Image img = new Bitmap(width, height);

int corna = rnd.Next(1000); // zufälliger wert von 0 bis < 1000
int cornb = rnd.Next(1000);
int side = rnd.Next(1000);

using (Graphics gfx = Graphics.FromImage(img)) {
	for (int i = 0; i < height; i++) {
		for (int j = 0; j < width; j++) {
            
        	int x = corna + (i * side / 100);
         	int y = cornb + (j * side / 100);
         	int c = x * x + y * y;

            // c % 3, gibt den Rest aus der Divison c / 3 zurück, bspw. 4 % 3 = 1 (4 / 3 = 1 Rest 1)
			SolidBrush brush = (c % 3) switch {
				0 => new SolidBrush(Color.Black),
				1 => new SolidBrush(Color.White),
				_ => new SolidBrush(Color.Gray), // default
			};
            
			gfx.FillRectangle(brush, i, j, width, height);
		}
	}
}

img.Save(@$"wallpaper{corna}_{cornb}_{side}.Png", ImageFormat.Png);

Zugegeben der Code brauch zumindest bei mir ziemlich lange bei der Ausführung.
